### **Brainstorming Session**

The team organized a structured brainstorming session to generate and refine project ideas. Several creative techniques were used to encourage participation and innovation.

#### A. Mind Mapping Technique

The team used mind mapping to visually organize related ideas and identify connections between problems, users, and possible solutions.

Central Problem: Difficulty Managing Medications for Elderly Patients

Related Challenges Identified:

* Forgetting medication times  
* Multiple prescriptions from different hospitals

Possible Solutions Generated:

* Medication reminder system

* Simple Arabic-first interface  
* Alert notifications for missed doses  
* Family support notifications (future enhancement)

Result: Mind mapping helped the team narrow the focus toward healthcare-related solutions with strong social impact.

#### **B. SCAMPER Framework**

The SCAMPER method was used to improve and innovate ideas by analyzing existing solutions.

| SCAMPER Element | Application to the Idea |
| :---- | :---- |
| Substitute | Replace paper medication schedules with digital reminders |
| Combine | Combine medication tracking with adherence monitoring |
| Adapt | Adapt reminder systems used in fitness apps for healthcare |
| Modify | Simplify interfaces specifically for elderly users |
| Put to Another Use | Use notification systems for healthcare safety alerts |
| Eliminate | Remove unnecessary complex features from the MVP |
| Reverse | Design the app around user simplicity instead of feature complexity |

Outcome: The SCAMPER process helped refine the MVP into a focused, user-friendly solution instead of a complex healthcare platform.

## **4\. Decision and Refinement**

### **Final MVP Selection**

After completing the brainstorming, research, and evaluation phases, the team held a final discussion session to review the scoring results and determine the most suitable MVP idea. Following team discussions and a voting process, the team officially selected the:

Smart Medication Management App for Elderly Patients as the final MVP concept for the project.

The decision was based on:

* High evaluation scores across all criteria  
* Strong real-world impact  
* Alignment with team interests and technical skills  
* Clear MVP scope suitable for the project timeline  
* Opportunities for future scalability and innovation

### **Refinement of the Selected MVP**

After selecting the idea, the team refined the concept to clearly define the problem, target users, core features, and expected outcomes.

#### A. Problem Statement

Many elderly patients struggle to manage medications effectively due to:

* Multiple prescriptions from different hospitals or clinics  
* Forgetting medication schedules  
* Difficulty organizing daily doses  
* Risk of medication duplication or conflicts  
* Lack of a centralized medication management system

These challenges can lead to:

* Missed doses  
* Incorrect medication usage  
* Health complications  
* Reduced treatment effectiveness

The project aims to simplify medication management and improve adherence through a user-friendly digital solution.

#### B. Target Audience

The primary target users are:

Elderly Patients \- Users who need assistance managing:

* Daily medication schedules  
* Multiple prescriptions  
* Reminder notifications

Secondary Users (Future Expansion):

* Family members assisting elderly relatives  
* Caregivers  
* Healthcare providers (future enhancement)

#### C. Key Features of the MVP

The team identified a focused set of features to keep the MVP achievable while still delivering meaningful value.

**Core Features:**

1. Medication Management  
   * Add medications manually  
   * Store medication name, dosage, and schedule  
   * View current medication list  
2. Reminder Notifications  
   * Scheduled alerts for medication times  
   * Daily reminder system  
   * Missed dose notifications  
3. Daily Dashboard  
   * Overview of today's medications  
   * Medication completion tracking  
   * Simple progress indicators  
4. User-Friendly Interface  
   * Large buttons and readable text  
   * Arabic-first design  
   * Simple navigation suitable for elderly users

#### D. Expected Outcomes

The team expects the MVP to achieve the following outcomes:

User Benefits:

* Improved medication adherence  
* Reduced missed doses  
* Easier organization of prescriptions  
* Better daily medication tracking

Technical Outcomes:

* Successful implementation of a healthcare-focused application  
* Practical experience in frontend and backend integration  
* Experience designing accessible user interfaces

Future Opportunities:  
The MVP can later evolve to include:

* Hospital or pharmacy integrations  
* Drug interaction detection  
* AI-powered medication insights  
* Family monitoring features  
* Cloud-based medical history synchronization

#### **E. Challenges Identified During Refinement**

During refinement, the team identified several important challenges:

| Challenge | Mitigation Approach |
| :---- | :---- |
| Designing for elderly accessibility | Use simple layouts, large fonts, and minimal complexity |
| Notification reliability | Prioritize stable reminder functionality |
| Time limitations | Focus only on core MVP features |
| Healthcare data complexity | Avoid external integrations during MVP stage |

### **Final Refined MVP Summary**

The final MVP is a simplified healthcare application designed to help elderly patients manage medications safely and efficiently through reminders, medication tracking, and an accessible user experience.

The refined concept balances:

* Real-world usefulness  
* Technical feasibility  
* Innovation potential  
* Clear development scope

This refinement process provided the team with a well-defined project direction and a strong foundation for the planning and development stages.
